Het verschil tussen internet en het World Wide Web

Het internet is de technische infrastructuur waarop het WWW berust; servers, kabels, draadloze verbindingen (satelliet, telefoon), computers. Het WWW bestaat uit alle opgeslagen informatie die middels het internet bereikbaar is.
Mijn vergelijking:

  • Het internet is als een bibliotheek: het gebouw, de boekenkasten
  • Het WWW zijn de te lenen boeken en andere materialen.

2.
HTML (hypertext Markup Language) is een taal waarmee je een pagina aan indelen en deels kunt opmaken. D.m.v. het meegeven van coderingen aan paginaelementen kan er worden meegegeven hoe een internetbrowser een internetpagina moet tonen wat de structuur betreft. De tekst wordt opgedeeld in titels, paragrafen en andere elementen. Voor een deel hebben deze alle een default styling. HTML is niet dynamisch.

CSS (Cascade Style Sheet) geeft de browser mee hoe elementen van een internetpagina opgemaakt moeten worden. Styling als kleur, textopmaak en grootte van een element kan via CSS worden bepaald. De browser interpreteert een CSS van boven naar beneden. Dus de laatste styling mbt tot een element die een browser tegenkomt wordt gebruikt. Als op regel 1 staat dat een woord vetgedrukt moet zijn, en op regel twee staat dat datzelfde woord niet vetgedrukt moet zijn, dan wordt het woord niet vetgedrukt weergegeven. Javascript is een programmeertaal waarmee elementen van een internetpagina dynamisch kunnen worden gemaakt. Het voegt interactiviteit toe aan de webpagina. In tegenstelling tot HTML en CSS kan het ervoor zorgen dat er iets gebeurd als een gebruiker ergens op klikt, een bepaalde toets indrukt. Of dat een gebruiker iets kan invullen. Javascript wordt in de browser uitgevoerd en is dus client-side.

Een goede HTML strucuur maakt de webpagina leesbaarder. Maar ook: een goede HTML-strucuur maakt aan zoekmachines duidelijk wat voor een inhoud de webpagina heeft. Dit is belangrijk om gevonden te worden door diegenen die jou informatie zoeken.

Een inline block neemt alleen de ruimte in die het nodig heeft. Een block-element pakt de hele regel.
Een block-element begint altijd op een nieuwe regel. Een inline-element hoeft dat niet.

8. Nesting is het plaatsen van elementen binnen andere elementen.

9.
Bij HTML semantics wordt de opmaak stijl van elementen losgekoppeld van de structurele betekenis. De structurele betekenis staat voorop. Vervolgens kunnen we deze eventueel nog stylen.
Voor mensen is een geode structuur belangrijk, maar voor zoekmachines en webcrawlers nog veel meer. Deze kunnen niet door een styling heenkijken. We moeten dus de elementen gebruiken die bij een bepaalde structurele inhoud passen. Voor een titel pakken we dus een header-element. En niet een paragraaf-element met een groter lettertype.
